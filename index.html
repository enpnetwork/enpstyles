enpresources

Resource gem for ENP Network and its sister sites.

Assets

Please see the assets guide to see the styles defined in this gem: ENP Style Guide

Usage

Add the following to your Gemfile: 

source "http://gems.enpnetwork.com"
gem 'enpresources'

Because this gem uses SCSS mixins, your CSS manifest file (application.css, for example) will need to be renamed to application.scss. Change all require statements to 

@import 'somefile';
@import 'some/other/file';

Add the following directive to your new SCSS manifest file at the top of the file: 

@import 'enpresources';

To add IE 8 or less support, add the following to your layout (using HAML for this example): 

/[if lt IE 8]
  = stylesheet_link_tag 'enpresources/ie'
  = javascript_include_tag 'enpresources/ie'

Additionally, you can require any of the SCSS files or manifests individually by importing the individual files in your app's manifest file.

There are also several vendored libraries included. Please see the section on Vendored Javascript and CSS Libraries below.

Documenting SCSS Code

Comment and document as much as possible! The stylesheets are documented using StyleDocco, which can be found at http://jacobrask.github.com/styledocco/.

Examples of markup can be found there as well.

Generating Documentation

You can prep the /docs folder for documentation by running: 

rake documentation:css

at the root of the gem. This rake task simply clears the /docs folder out and copies image assets over. It will then give you instructions on how to generate the documentation with StyleDocco (which are also below).

If you just want to generate the documentation without preparation, run the following command: 

styledocco -n 'ENP Resources' app/assets/stylesheets

and the documentation will be generated in the /docs folder.

Additionally, you can copy the /hooks/post-commit file to /.git/hooks, and make sure it's executible by running chmod +x ./post-commit. It will then execute the documentation generation after every push to remote.

Rails, Sprockets, and the Asset Pipeline

As a note, the following files use the image-url Sprockets helper to parse the asset path:

  • /app/assets/stylesheets/base/layout.scss
  • /app/assets/stylesheets/base/navigation.scss
  • /app/assets/stylesheets/base/tables.scss
  • /app/assets/stylesheets/components/buttons.scss
  • /app/assets/stylesheets/components/boxes.scss
  • /app/assets/stylesheets/components/tabs.scss
  • /app/assets/stylesheets/components/toolbars.scss

The following file uses the stylesheet-url Sprockets helper:

  • /app/assets/stylesheets/mixins/css3.scss

This is necessary for having assets on a remote server, such as Amazon S3. In order to generate documentation for these files, each file must have SCSS variables set at the top of the document for each image used. When documentation is generated, they can be overridden by uncommenting out variables using the url() CSS definition, instead of the image-url() sprockets helper. See any of the files above for more details.

This file (README.md) will be added automatically as the index.html file for the documentation.

Vendored Javascript and CSS Libraries

The following libraries are vendored in this application:

  • Ajax Autocomplete for jQuery, version 1.1.3
  • jQuery 1.8.2
  • jQuery ColorBox, version 1.3.20.2
  • jQuery jCrop, version 0.9.10
  • Twitter Bootstrap Popovers and Tooltips, version 2.1.1

You can include any of these in your files as follows (using HAML for this example): 

/ For Ajax Autocomplete:
= stylesheet_link_tag 'autocomplete/autocomplete'
= javascript_include_tag 'autocomplete/autocomplete'

/ For jQuery:
= javascript_include_tag 'jquery-1.8.2'

/ For jQuery ColorBox:
= stylesheet_link_tag 'colorbox/colorbox'
= javascript_include_tag 'colorbox/jquery.colorbox'

/ For jQuery jCrop:
= stylesheet_link_tag 'jcrop/jcrop'
= javascript_include_tag 'jcrop/jcrop'

/ For Twitter Bootstrap Popovers and Tooltips:
= stylesheet_link_tag 'bootstrap/bootstrap'
= javascript_include_tag 'bootstrap/bootstrap'
